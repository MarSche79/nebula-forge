Trust boundaries

Four checkpoints between browser & API.

The portal is publicly reachable but the API is internal-only. Even the portal can't talk to the API without proving who it is — and what user it's acting on behalf of.

1 · Edge
Public ingress
HTTPS only, allowInsecure=false. Anyone may load the landing page or careers section.
unauthenticatedClientAction: AllowAnonymous
2 · Page
Next.js middleware
/command-center, /dashboard, /hr/* require X-MS-Client-Principal. Missing → 307 to /.auth/login/aad.
azure/portal/src/middleware.ts
3 · Proxy
Catch-all proxy
Allow-listed paths only. Strips inbound x-ms-*, x-forwarded-*, authorization, cookie, x-proxy-auth before re-injecting trusted ones.
/api/[...path]/route.ts
4 · API
requireAuth middleware
Verifies x-proxy-auth shared secret + decodes principal from x-ms-client-principal. Constant-time comparison.
azure/api/src/auth/jwt.ts

One exception: POST /api/applications (the public Apply form) is intentionally anonymous — guarded by zod validation, 24 KB cvText cap, jobId allow-list, per-IP rate limit (5/h), and 30 s (email, jobId) deduplication. Never logs raw CV text.

The HR screening pipeline

Two AI agents, in serial, on every CV.

Every CV submitted at /careers/[id] flows through Azure OpenAI twice: an Interviewer Agent scoring fit, then an HR Manager Agent recommending next steps. Anything blocked by the content filter becomes a Defender for AI alert.

1

📨 Public submit

Browser uploads PDF → client-side pdfjs-dist extracts text → POST /api/applications.

{ name, email, jobId, cvText, coverNote, fileName }
2

🎯 Interviewer Agent

gpt-4o-mini in JSON mode. Scores match, lists strengths & gaps, suggests interview focus.

{ "matchScore": 90, "verdict": "Recommend", "strengths": [...], "gaps": [...] }
3

🧑‍💼 HR Manager Agent

Second gpt-4o-mini call reading the Interviewer's output. Outputs a recommendation + risk flags.

{ "recommendation": "Proceed to Tech.", "rationale": "...", "nextSteps": "...", "riskFlags": [] }
4

🐘 Persist to Postgres

Insert into candidates table tagged source='web'. HR can view, decide, delete.

UPDATE candidates SET decision = 'Hired', status = 'Screened' WHERE id = $1;

🛡️ When the content filter blocks a prompt …

The pipeline catches OpenAI's content_filter / ResponsibleAIPolicyViolation error and stores the application with status = "Flagged", threatDetected = true, and the filtered category names. The row appears in /hr/threats. Defender for AI raises the corresponding alert in Defender for Cloud → Security Alerts (15-30 min propagation).

The other workforce

Nine specialist agents, one Master Agent.

The Master Agent in ca-api exposes nine ask_<agent> function tools to OpenAI. When the model picks one, the API does a second OpenAI completion scoped to that agent's MCP toolset, lets the model fill in the arguments, and dispatches the call.
